📊 领域全景 · 专家视角导读

Attention 机制是 Transformer 架构的核心，但其 O(n²) 的时间与空间复杂度在处理长序列时成为严重瓶颈。这个领域的本质问题很直接：如何在保持标准 softmax attention 表达能力的同时，将计算复杂度降到线性或近线性？

当前主流共识是三条技术路线并行发展：（1）IO 感知的精确计算路线（以 FlashAttention 系列为代表），不改变数学等价性，通过优化 GPU 内存访问模式实现 2-4x 加速；（2）稀疏注意力路线，通过动态或静态地跳过不重要的注意力块，将实际计算量减少 50-95%；（3）线性近似路线，用核函数或状态空间模型替代 softmax，实现 O(n) 复杂度，但在某些任务上仍有精度损失。

2024-2025 年的关键争议在于：稀疏 vs 线性，哪个更能兼顾质量与效率？近期 DeepSeek 的 Native Sparse Attention 和清华的 SLA（Sparse-Linear Attention）给出了一个融合方向——将稀疏和线性注意力统一到一个可训练框架中。另一个热点是将这些加速方法从 LLM 推广到 Diffusion Transformer（视频生成），序列更长，瓶颈更突出。

新手建议：先读 FlashAttention 三部曲理解 IO 优化基础，再深入 Native Sparse Attention 和 SLA 了解最新融合路线。

✅ 通过 IO 感知的分块算法减少 HBM 访问次数⚠️ 遗留：仅优化 IO，未降低 FLOPs⭐ 8k 引用
