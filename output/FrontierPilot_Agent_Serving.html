📊 领域全景 · 专家视角导读

Agent Serving System Optimization 是 2024-2026 年快速崛起的系统研究方向，核心问题是：如何高效地为 LLM 驱动的智能体（Agent）工作负载提供推理服务？

传统 LLM serving 系统（如 vLLM、SGLang）假设的是「单次请求→单次生成」的交互模式，但 Agent 工作流本质是多轮、异步、有状态的——涉及 tool calling、多模型编排、动态控制流和长时运行状态。这些特征让现有 serving 系统面临三类新挑战：(1) 动态调度——Agent 的执行路径由模型输出决定，无法提前预测；(2) 状态管理——长对话 + 工具执行结果需要持久化的逻辑状态，而非仅是 KV cache；(3) 异构组件——Agent pipeline 中可能混合不同模型（planner + executor）、不同精度、甚至非 LLM 组件。

目前该领域呈现两条演进路线：一条是从传统 LLM serving 向上扩展，增加对 Agent 工作流的支持（如 Nalar 的 futures 机制、SDN-inspired serving 框架）；另一条是从 Agent 框架向下优化，嵌入 serving-aware 的资源管理。2025-2026 年的关键共识是：Agent serving 需要「工作流与执行解耦」的架构——将 Agent 逻辑（Python 代码）与系统调度（资源分配、请求路由）显式分离。

新手建议：(1) 先读 vLLM/PagedAttention 理解 LLM serving 基础；(2) 重点读 Nalar（arXiv 2601.05109），它最完整地定义了 Agent serving 的核心抽象；(3) 关注 FlashInfer 等 kernel 级优化——它们是底层性能的决定因素。
